The <figcaption> element in HTML is used to provide a caption or description for a <figure> element. I learned that it must be placed inside the <figure> tag and is often used to describe images, illustrations, or other visual content. It improves accessibility and clarity by giving context to the media it accompanies. I also found out that <figcaption> can be positioned either before or after the content inside <figure>, which gives flexibility in styling and layout. Adding it to my website helped me create more informative and structured content around images.

While exploring CSS, I learned about the animation property and @keyframes, which allow for smooth and dynamic animations on a webpage. The animation property is a shorthand that combines multiple animation-related properties, such as duration, timing function, and iteration count. Meanwhile, @keyframes defines the steps of an animation by specifying how an element should change at different points in time. I experimented with animating a button to change colors and scale up when hovered over. I found that adding ease-in-out timing and adjusting animation-delay made the effect feel more natural. Using animations helped make my website feel more interactive and visually engaging.
